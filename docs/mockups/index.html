Voxglass — plan mockups

Five plans written 22 July 2026. Each mockup file is self-contained — open it directly, no server needed. Phone frames are 390 × 844 pt; the watch frames are true 45 mm (198 × 242 pt) shown at 1.5×.

Plan 1
Unified Book Page — Now Playing ∪ Book Details
One screen replaces both. Author and narrator always visible, description clamped to one line with “Show more” above the chapters, every full-width button reduced to a 40 pt icon, AirPlay added, long tail behind ⋯.
docs/UNIFIED_BOOK_PAGE_PLAN.md
Plan 2
Voxglass for Apple Watch — standalone player
Nine screens: now playing, library, local + remote search, book page with a readable description, chapters, on-watch storage, speed & sleep, complications.
docs/WATCH_APP_PLAN.md
Plan 3
App Intents, Live Activity & Siri
Lock-screen Live Activity, Dynamic Island, Siri phrases that never open the app, Shortcuts / Spotlight / Action button, widgets and Control Center controls — all through one intent layer.
docs/INTENTS_LIVE_ACTIVITY_SIRI_PLAN.md
Plan 4
Voxglass Studio — paid AI-narrated titles
$1.99–$2.99 non-consumable IAPs for classics LibriVox has never recorded. Store catalogue and a product page with a free chapter-1 sample and synthetic-narration disclosure.
docs/AI_NARRATION_STORE_PLAN.md
Plan 5
Narration Markers, Loading Feedback & Tactile Taps
Solo Narration becomes a conservative book signal across lists, cards and detail pages. Remote imports show per-row or per-card loading state immediately, and tappable app actions get light tactile feedback.
docs/NARRATION_AND_TACTILE_FEEDBACK_PLAN.md
